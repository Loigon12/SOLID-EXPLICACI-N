Principio de Abierto/Cerrado (OCP)

El Principio de Abierto/Cerrado es uno de los pilares de SOLID en la programación orientada a objetos. Fue propuesto por Bertrand Meyer en 1988 y establece que las entidades de software deben estar abiertas para extensión pero cerradas para modificación.

¿Qué significa?

Significa que podemos añadir nuevas funcionalidades sin alterar el código existente. En lugar de modificar clases ya probadas, se crean nuevas implementaciones que extienden el comportamiento. Esto se logra mediante el uso de interfaces, herencia y patrones de diseño.

¿Cómo funciona?

El principio se aplica creando nuevas clases que extienden el comportamiento sin modificar el código base. Por ejemplo, tenemos una clase Saludo que define un método mostrar(). En lugar de cambiar esa clase cada vez que queremos un saludo diferente, creamos nuevas clases que extienden su comportamiento.

class Saludo {
    public void mostrar() {
        System.out.println("Hola!");
    }
}

class SaludoFormal extends Saludo {
    public void mostrar() {
        System.out.println("Es un honor dirigirme a usted. Confío plenamente en que la 
                            presente jornada esté transcurriendo bajo los más estrictos estándares de excelencia y provecho.");
    }
}

class SaludoInformal extends Saludo {
    public void mostrar() {
        System.out.println("¡Hablalo Calabazo, como me lo trata la laif mi papacho!");
    }
}

De esta manera, el sistema permite agregar nuevos tipos de saludos sin modificar la clase original Saludo. Solo se crean nuevas extensiones que cumplen el principio de abierto/cerrado.
